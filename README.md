## 项目介绍

### 实时字幕提词器

`live-teleprompter/` 是一个面向口播、录制、直播和演讲场景的浏览器端提词器应用。用户可以粘贴或保存稿件，在阅读时通过语音识别自动匹配当前朗读位置，并让稿件跟随朗读进度滚动。

核心能力：

- **语音跟读滚动**：基于浏览器语音识别能力监听朗读内容，自动匹配稿件位置并滚动到当前段落。
- **固定速度滚动**：在语音识别不可用或不需要跟读时，可以按设定速度自动滚动。
- **稿件管理**：支持本地保存、打开、删除稿件，适合反复使用常用口播脚本。
- **阅读控制**：支持字号调整、灵敏度调整、提前量设置、上一段/下一段手动同步和回到开头。
- **全屏模式**：提供更适合录制和演讲的沉浸式阅读界面。
- **调试与诊断**：内置麦克风权限检测、音量测试、模拟识别文本、匹配置信度和事件日志，方便排查语音识别问题。

运行说明：

- 该应用是纯前端静态页面，发布目录内包含 `index.html`、主应用脚本、样式文件和匹配算法 worker。
- 语音跟读依赖浏览器的 `SpeechRecognition` / `webkitSpeechRecognition` 能力，不同浏览器支持情况可能不同。
- 麦克风权限和语音识别通常需要 HTTPS 或安全上下文；GitHub Pages 访问地址满足这一要求。
- 稿件和偏好设置保存在浏览器本地存储中，不需要后端服务。

## 发布方式

该仓库面向静态托管，不包含应用源码、依赖安装流程或本地构建脚本。推荐流程是：

1. 在对应项目仓库中完成构建。
2. 将构建产物复制到本仓库的目标目录。
3. 确认入口文件和资源路径使用相对路径，保证 GitHub Pages 子路径下可以正常加载。
4. 提交并推送到 GitHub，由 GitHub Pages 提供访问。

## 维护规范

- 每个发布目标使用独立目录，避免不同应用的资源互相覆盖。
- 保留构建工具生成的 hash 文件名，便于缓存失效和版本追踪。
- 更新资源时同步检查入口 `index.html` 中引用的文件是否存在。
- 如新增静态页面，请在上方“当前发布内容”表格中补充路径、说明和访问地址。
- 不在本仓库存放源码级依赖、临时文件、密钥或本地环境配置。

## 本地预览

如果只需要快速检查静态文件，可以在仓库根目录启动任意静态文件服务器：

```bash
python3 -m http.server 8000
```

然后访问：

```text
http://localhost:8000/live-teleprompter/
```
